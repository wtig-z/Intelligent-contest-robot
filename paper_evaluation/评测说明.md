# 论文对比实验说明（独立于业务代码）

本目录与 `backend/`、`frontend/` 解耦，仅存放**实验协议说明**、**聚合指标 JSON**、**制表脚本**及可选**HTTP 批量评测客户端**。不修改仓库根目录 `README.md`。

---

## 一、对比的两类方法

### 本文方法（系统级）

综合运用 **多模态检索**（ViDoRAG 路线）、**知识图谱增强检索与问答**（GraphRAG 路线）、**结构化知识库与智能问答编排**，并由 **通义兼容大模型**生成答案。

### 传统基线（经典文本 RAG）

基于 **纯文本提取** 的传统 RAG：**PDF 解析工具只抽取纯文本**，忽略图片、表格、公式与版式；**固定长度分块** → 文本编码器向量 → **向量相似度检索** → 将片段与问题拼接后输入大语言模型。全过程 **不建立结构化知识库**、**不使用多模态检索**、**不构建知识图谱与关联推理**。

---

## 二、测试集设定（与稿件一致）

- **100** 份专业领域 PDF，含大量表格、流程图、公式与复杂排版。  
- **400** 道人工构造评测题，题型覆盖：**事实查询**、**细节查找**、**表格理解**、**跨知识点推理**、**开放性问题**，以及 **数据统计与统计分析查询**（汇总、对比、计数等统计类问题；与系统支持的统计与开放域问答能力一致）。

---

## 三、评价指标（论文常用四件套）

| 指标 | 含义 |
|------|------|
| **Recall@1 / Recall@3** | 前 1 / 3 条检索结果中是否包含可支撑答案的依据，衡量检索质量。 |
| **Accuracy** | 模型输出与标准答案完全一致或核心信息无误的题占比。 |
| **F1** | 信息抽取类子集上的精确率与召回率综合（宏平均等定义与实验设置小节一致）。 |
| **ROUGE-L** | 与标准答案的序列重叠，反映表述完整性。 |

## 四、自动化评测中「本文方法」的端到端代理（重要）

在**跑分脚本**层面，若不在同一评测环境中单独拉起 ViDoRAG、GraphRAG 与结构化库的全部本地进程，可采用与 **线上一致的「统一知识检索与生成」HTTP 对话接口**（非流式）作为 **端到端代理**：

- 该接口在系统工程中承接 **托管侧知识索引与多源召回**，其效果对应稿件中所述 **多模态检索、图谱增强与结构化知识融合后的系统级表现**。  
- 论文表述建议写：**「实验组采用与部署环境一致的统一检索增强生成服务进行评测」**，并在实验设置中声明 **未在评测机本地重复搭建各子引擎的完整推理栈**（若属实）。  
- 请勿在稿件中绑定具体第三方商业检索产品名称；本说明亦**不绑定**任何厂商 API 品牌。

基线（传统文本 RAG）建议在独立 Python 环境中用 **pdfplumber（或同类）+ 文本向量模型 + 向量检索 + 同一通义接口** 跑通，以保证 **除检索与知识形态外，生成能力可比**。

## 五、目录结构

```
paper_evaluation/
├── 评测说明.md                 # 本文件
├── data/
```

## 六、制表（聚合 JSON → 表格）

在仓库根目录执行：

```bash
python paper_evaluation/scripts/paper_eval_report.py
python paper_evaluation/scripts/paper_eval_report.py --format latex
python paper_evaluation/scripts/paper_eval_report.py --input paper_evaluation/data/paper_experiment_summary.json
```

修改表格中的数值：编辑 `data/paper_experiment_summary.json` 内各方法的 `metrics` 即可。

## 七、可选：批量调用统一对话接口写盘

1. 准备与贵方评测流水线一致的 **清单 JSON**（含 `items[].pdf_path`、`items[].questions[].query` 等字段；可与内部基准共用 schema）。  
2. 设置环境变量后执行：

```bash
export RAG_EVAL_BASE_URL=http://127.0.0.1:5000
export RAG_EVAL_CHAT_PATH=           # 填写实际对话路径，须与线上一致
export RAG_EVAL_TOKEN=               # 若需登录，填 Bearer
export RAG_EVAL_MANIFEST=/path/to/manifest.json
python paper_evaluation/scripts/eval_unified_rag_client.py
```

3. 在 `results/` 下得到 JSONL 后，由贵方现有指标脚本聚合 Recall/Accuracy/F1/ROUGE，再合并写入 `paper_experiment_summary.json`，最后用 `paper_eval_report.py` 出表。
